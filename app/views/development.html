EngineJS was designed to be a simple MVC framework with two-way data binding.

The aim was to bring the best open source software into one place so the developers could create virtually any web application with one installation.
There are 3 ways of rendering data.
1. Using handlebars "Handlebars.js". Though EngineJS has add a couple of extra helper (eg switch)

                    {{handlebars}}
This is fine with the data is unlikely to change or you want some logic.

Result

{{title}}
{{{entry.name}}} {{#if entry.active}} Active {{/if}} {{#each list}}
{{name}}
{{/each}} {{#switch state}} {{#case "page1" "page2"}}page 1 or 2{{/case}} {{#case "page3"}}page3{{/case}} {{#case "page4"}}page4{{/case}} {{#case "page5"}} {{#switch s}} {{#case "3"}}s = 3{{/case}} {{#case "2"}}s = 2{{/case}} {{#case "1"}}s = 1{{/case}} {{#default}}unknown{{/default}} {{/switch}} {{/case}} {{#default}}page0{{/default}} {{/switch}}
2. EngineJS notation. Using {% express %} or {= express =} 
                    {% notation %}
So what is the difference.

{% creates a data bind element () that will be updated if the associated model data is changed

{= is a literal render of data but unlike "handlebars" which uses {{{ }}}, it is a javascript evaluation (eval function) of the expression.

Result


developmentModel.qty = {= developmentModel.qty =}

developmentModel.qty * 2 = {% developmentModel.qty * 2 %}

developmentModel.getQty() * 3 = {% developmentModel.getQty() * 3 %}
3. Using element attributes. 
                    {= developmentModel.binding =}
The "en-model" is optional. If not provide EngineJS will use the controller model.
