Engine JS includes elements, these are components except they only get created then the element tag is found.

So why elements
  • An element can be created more than once in your app. Using the same code
  • Passing attributes to the element tag allows each instance to be unique
  • Two-way data binding still applies
Elements are registered.

$engine.registerElement("myElementName",model, view, options);

To use the element simply include the element name as a HTML tag.

<myElementName></myElementName>

or to pass attributes use...

<myElementName index="0"></myElementName>
<myElementName index="1"></myElementName>

registerElement parameters:

NAME
Any user definded name that will be later used in the HTML of a view.
MODEL
A standard EngineJS model. If null is pass an empty model is created. Any HTML element attributes are added to the model so they can used in the newely created component.
VIEW
A standard EngineJS view. If a string (containing HTML) is pass an view based on the string is created.
OPTIONS
A JSON object including and methods.
{
    methods: {
        onrender:function()
        {

        }
    }
}

We don't recomment passing an id in the options JSON. A unique one will be issued for each instance.
